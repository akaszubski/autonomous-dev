# CHANGELOG Format

Follow Keep a Changelog (keepachangelog.com):

```markdown
## [Unreleased]

### Added
- New features

### Changed
- Changes to existing functionality

### Fixed
- Bug fixes
```

## Standard Categories

All changes should be categorized using semantic versioning principles:

- **Added**: New features
- **Changed**: Changes to existing functionality
- **Deprecated**: Soon-to-be removed features
- **Removed**: Removed features
- **Fixed**: Bug fixes
- **Security**: Security vulnerability fixes

## Format Standards

- Use present tense ("Add" not "Added")
- Be concise but descriptive
- Link to issues/PRs when relevant
- Keep entries under Unreleased section until release
- Version releases follow semantic versioning (MAJOR.MINOR.PATCH)
